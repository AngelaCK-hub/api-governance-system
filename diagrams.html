Figure 2

Entity-Relationship (ER) Diagram

📡 APIS
PKid INTEGER
name TEXT NOT NULL
description TEXT
allowed_usage INTEGER
cost_per_call REAL
👥 USERS
PKid INTEGER
name TEXT NOT NULL
role TEXT NOT NULL
📊 ACCESS_LOGS
PKid INTEGER
FKuser_id → users(id)
FKapi_id → apis(id)
timestamp DATETIME
usage_count INTEGER
🚨 ALERTS
PKid INTEGER
type TEXT NOT NULL
FKapi_id → apis(id)
FKuser_id → users(id)
status TEXT
message TEXT
timestamp DATETIME

🔗 Relationships

• One User ——→ has many AccessLogs (1 : N)
• One API ——→ has many AccessLogs (1 : N)
• One API ——→ has many Alerts (1 : N)
• One User ——→ has many Alerts (1 : N, optional)

Figure 4

Alert Engine — Processing Flowchart

▶ Start: check_and_generate_alerts()
↓
Connect to database
↓
Fetch all APIs from database
↓
🔁 FOR each API:
↓
Calculate total_usage = SUM(access_logs)
↓
Calculate ratio = total_usage ÷ allowed_usage
↓
Delete old alerts for this API
↓
ratio ≥ 1.0
↓
🔴 Create CRITICAL Alert
ratio ≥ 0.75
↓
⚠️ Create WARNING Alert
ratio < 0.75
↓
✅ No alert needed
↓
Save changes to database
↓
■ End
